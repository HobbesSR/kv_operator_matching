# Experiment Precommit Template

Keep this short enough that it is used.

Required:

- `Hypothesis`:
- `Surface`:
  prompts / layers / budgets / regimes / query-bank density
- `Primary metric`:
- `Mechanism expectation`:
- `Promotion rule`:
  distinguish broad-surface promotion from slice-only follow-up interest
- `Kill rule`:

Optional:

- `Artifact path`:
- `Pivot condition / follow-up trigger`:
  if the main hypothesis fails but a different broad pattern is strong, treat
  that as a lead for a new precommit, not as a success for the current one

Use this as a seatbelt, not a steering lock:

- prevent retroactive success redefinition
- prevent slice wins from being promoted as broad results
- allow pivots, but require naming the pivot explicitly
